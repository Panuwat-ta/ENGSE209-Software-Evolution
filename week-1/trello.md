#### 📌 TASK-05: Negative Value Input Prevention
*   **ความสำคัญ:** สูงสุด (Critical) | **ประมาณการเวลา:** 2 ชั่วโมง
*   **ผู้รับผิดชอบ:** Developer
*   **รายละเอียด:** บล็อกและจำกัดขอบเขตค่าข้อมูลตัวเลขไม่ให้เป็นลบ เพื่อความสมเหตุสมผลทางการค้าและสต็อกสินค้า
*   **รายการย่อยที่ต้องทำ (Sub-tasks Checklists):**
    *   [ ] บล็อกไม่ให้ระบุราคาสินค้า (Price) ติดลบตอนเพิ่ม/แก้ไขสินค้า (ต้องมีค่า `>= 0.0`)
    *   [ ] บล็อกไม่ให้ระบุจำนวนสินค้า (Quantity) ติดลบตอนเพิ่มสินค้าเข้าสต็อก (ต้องมีค่า `>= 0`)
    *   [ ] บล็อกไม่ให้ระบุจำนวนตัดยอดออกสินค้า (Out Quantity) มีค่าติดลบหรือเท่ากับ 0 (ต้องมีค่า `> 0`) เพื่อป้องกันบั๊ก "การตัดยอดติดลบเพื่อแอบเพิ่มสต็อกสินค้า"
*   **เงื่อนไขความสำเร็จ (Definition of Done - DoD):**
    *   การกรอกค่าติดลบใดๆ จะต้องแสดงคำเตือนทางระบบ เช่น "กรุณากรอกตัวเลขที่มากกว่าศูนย์เท่านั้น" และไม่ยอมรับค่านั้นเข้าระบบ

*   **ความสำคัญ:** สูง (High) | **ประมาณการเวลา:** 3 ชั่วโมง
*   **ผู้รับผิดชอบ:** Tech Lead / Developer

    *   สามารถทนทานต่อสถานการณ์จำลองที่ระบบถูกขัดจังหวะกระทันหัน (Simulated Crash) ระหว่างเขียนฐานข้อมูล โดยไฟล์เดิมต้องยังอยู่สมบูรณ์
