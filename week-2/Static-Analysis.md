## 2. ประเด็นที่ตรวจพบจาก Pylint (Linter Findings)

Pylint ได้ตรวจพบข้อควรแก้ไข (Warning/Convention) ตามมาตรฐาน PEP 8 แบ่งออกเป็นหมวดหมู่หลักดังนี้:

### 2.1 ปัญหาโครงสร้างและความเรียบร้อยของโค้ด (Formatting & Conventions)
* **C0303 (Trailing whitespace):** พบช่องว่างส่วนเกินท้ายบรรทัดที่ไม่มีการใช้งาน (บรรทัดที่ 35, 42, 49, 58, 75, 81, 87, 91)
* **C0301 (Line too long):** มีบรรทัดที่ยาวเกิน 100 ตัวอักษร (บรรทัดที่ 40 ยาว 119 ตัวอักษร)
* **C0304 (Final newline missing):** ขาดการเว้นบรรทัดใหม่บรรทัดสุดท้ายของไฟล์ (บรรทัดที่ 99)

### 2.2 ปัญหาด้านการตั้งชื่อและการจัดทำเอกสาร (Naming & Documentation)
* **C0114 (Missing module docstring):** ขาดคำอธิบายวัตถุประสงค์ของโมดูลที่ด้านบนสุดของไฟล์
* **C0116 (Missing function docstring):** ฟังก์ชัน `load()`, `save()`, และ `main()` ขาด Docstring เพื่ออธิบายอาร์กิวเมนต์และการทำงาน
* **C0103 (Invalid name):** ตัวแปรโกลบอล `db` ไม่ได้ใช้ตัวพิมพ์ใหญ่ทั้งหมด (ตามมาตรฐาน PEP 8 ควรใช้ตัวพิมพ์ใหญ่ทั้งหมดสำหรับ Constant เช่น `DB` หรือ `DB_PATH`)

### 2.3 ปัญหาโครงสร้างโค้ดและการขยายระบบ (Refactoring & Design Warning)
* **R0912 (Too many branches):** ฟังก์ชัน `main()` มีการแตกสาขาการตัดสินใจมากเกินไป (พบ 17 สาขา จากมาตรฐานกำหนดไม่เกิน 12 สาขา) ทำให้วิเคราะห์การทำงานของโค้ดได้ยาก
* **R0915 (Too many statements):** ฟังก์ชัน `main()` มีคำสั่งทำงานในฟังก์ชันมากเกินไป (พบ 55 คำสั่ง จากมาตรฐานกำหนดไม่เกิน 50 คำสั่ง)

## 3. ประเด็นที่ตรวจพบเกี่ยวกับความปลอดภัยและช่องโหว่ทางตรรกะ (Security & Logical Flaws)

จากการตรวจสอบซอร์สโค้ดแบบ Manual เพิ่มเติม พบช่องโหว่และข้อบกพร่องที่ร้ายแรงดังนี้:

### 3.1 การขาดการตรวจสอบประเภทข้อมูลนำเข้า (Unvalidated Input)
* **ปัญหา:** ที่บรรทัดที่ 46 และ 47 มีการแปลงอินพุตเป็น `int` และ `float` โดยตรง
  ```python
  c = int(input("Enter Qty: "))
  d = float(input("Enter Price: "))
  ```
* **ความเสี่ยง:** หากผู้ใช้งานป้อนตัวอักษรหรือค่าว่าง โปรแกรมจะแจ้งเตือนข้อผิดพลาด `ValueError` และเกิดการยุติการทำงานกระทันหัน (Runtime Crash)
* **แนวทางแก้ไข:** ควรเขียนฟังก์ชันดักจับ (Try-Except Block) เพื่อให้ผู้ใช้สามารถใส่ค่าใหม่ได้โดยโปรแกรมไม่ล่ม

### 3.2 ปัญหาการเข้ารหัสตัวอักษรขณะอ่าน/เขียนไฟล์ (Unspecified File Encoding)
* **ปัญหา:** ฟังก์ชันเปิดไฟล์อ่านและเขียนไม่ได้ระบุ Parameter `encoding`
  ```python
  with open(db, 'r') as f:
  with open(db, 'w') as f:
  ```
* **ความเสี่ยง:** เมื่อรันบนระบบปฏิบัติการที่แตกต่างกัน (เช่น Windows vs Linux/macOS) ระบบปฏิบัติการจะใช้ Encoding เริ่มต้นของเครื่อง (เช่น CP1252 บน Windows) ซึ่งอาจส่งผลให้การอ่านชื่อสินค้าที่เป็นภาษาไทยหรือสัญลักษณ์พิเศษเกิดการผิดเพี้ยน (Encoding Error หรืออ่านภาษาไทยเป็นอักษรต่างดาว)
* **แนวทางแก้ไข:** ระบุ `encoding='utf-8'` เสมอเมื่อเปิดไฟล์อ่านหรือเขียนข้อมูลภาษาไทย

### 3.3 การใช้ตัวแปรโกลบอลแชร์กัน (Global State Sharing)
* **ปัญหา:** มีการประกาศใช้ `global x` ในฟังก์ชัน `load()` เพื่อแชร์ข้อมูลสินค้า
* **ความเสี่ยง:** ทำให้การทำงานของแอปพลิเคชันผูกติดอยู่กับตัวแปรเดียว และมีสิทธิ์ถูกเขียนทับจากจุดอื่นโดยไม่ได้ตั้งใจ (Side Effects) ส่งผลให้ทำ Unit Test ได้ยากมาก
* **แนวทางแก้ไข:** เปลี่ยนมาใช้ Object-Oriented Programming (OOP) เก็บสถานะไว้ใน Attribute ของคลาสผู้จัดการระบบ

### 3.4 ข้อมูลสต็อกติดลบและราคาสินค้าติดลบ (Negative Quantity and Price)
* **ปัญหา:** ผู้ใช้สามารถป้อนจำนวนสต็อก (Qty) และราคา (Price) ติดลบในระบบได้ผ่านเมนู Add or Update
* **ความเสี่ยง:** ทำให้มูลค่าของคลังสินค้าติดลบ และตรรกะทางธุรกิจทำงานผิดพลาด
* **แนวทางแก้ไข:** เพิ่มเงื่อนไขตรวจสอบว่า `Qty >= 0` และ `Price >= 0` ก่อนจะอัปเดตข้อมูล
